**2.3.a LISP: The Control Plane**
LISP handles the where and who of the network by separating identity from location.

Core Concepts:
- EID (Endpoint Identifier): The actual IP or MAC of the host. This remains constant during mobility.
- RLOC (Routing Locator): The IP of the router/switch (VTEP) that the host is currently behind.
  
Key Network Roles:
- MS (Map-Server): The database "repository" that tracks EID-to-RLOC mappings.
- MR (Map-Resolver): Handles queries (Map-Requests) from routers looking for an EID's location.
- ITR/ETR (Ingress/Egress Tunnel Router): These devices (often called xTRs) perform the actual tunnel encapsulation and decapsulation.
- The "Pull" Mechanism: Unlike traditional routing where every router needs to know every route (push), LISP uses a demand-based (pull) model where an xTR only queries the MS/MR when it has traffic for a specific host.

**2.3.b VXLAN: The Data Plane**
VXLAN is the encapsulation method used to carry traffic over the LISP-calculated paths.

Technical Encapsulation:
- MAC-in-UDP: VXLAN wraps original Layer 2 Ethernet frames inside Layer 3 UDP packets (Destination Port 4789).
- Outer Header: Contains the Underlay IP addresses (RLOCs).
Scaling and Segmentation:
- VNI (VXLAN Network Identifier): A 24-bit field, allowing for ~16 million segments, far surpassing the 12-bit VLAN limit of 4,094.
- VTEP (VXLAN Tunnel Endpoint): The physical or virtual device that performs the tunneling. In SD-Access, the Fabric Edge nodes act as VTEPs.
Retention: Unlike pure LISP data planes, VXLAN retains the original Layer 2 header (Ethernet portion), which is critical for Layer 2 extensions across Layer 3 boundaries.

**Summary Notes**
These protocols are the foundation of SD-Access.

LISP + VXLAN:
- LISP is the directory service (Control Plane) telling the network which RLOC owns which EID.
- VXLAN is the transport (Data Plane) that creates the tunnels between those RLOCs.

Policy Plane (SGTs): Presentation should note that the VXLAN header in SD-Access also carries Scalable Group Tags (SGTs) for microsegmentation and security policy enforcement.
Underlay Requirement: The underlay must have full IP reachability (often via IS-IS or OSPF) before the LISP/VXLAN overlay can function.
